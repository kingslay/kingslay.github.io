Obc 类变量的实现方式

Dec 25th, 2013

obc中是没有类变量的。只能在实现类里增加一个静态变量。然后，调用initialize方法，来初始化。这个静态变量只能能在实现类里用，所以别的类也要用的话，那就要在头文件里增加一个类方法，用于返回这个类变量。(如果是在头文件增加静态变量的话，那就全局变量了。)

@interface PubUtil : NSObject
+ (void)loginDelegate;
@end


@implementation PubUtil
static LoginDelegate *loginDelegate;

+(void)initialize
{
    if (self == [PubUtil class])
    {
        loginDelegate = [[LoginDelegate alloc] init];
    }
}
@end

这样就可以在别的类调用这个类变量了。

[PubUtil loginDelegate]
//或是，但是用下面这种方式，xcode不会帮你提示有这个变量的。
PubUtil.loginDelegate

但是如果PubUtil有子类的话。那initialize就会调用两次。除非PubUtil的子类有实现了initialize方法，并且不能这样调用父类的initialize。如下

+(void)initialize{
    [super initialize];
}

所以为了解决这个问题。就像开头的写法if (self == [PubUtil class])，先判断下是不是当前类的类型。
